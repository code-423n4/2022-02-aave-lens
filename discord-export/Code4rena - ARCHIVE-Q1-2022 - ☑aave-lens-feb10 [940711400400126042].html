joshstevens19 02/09/2022 3:48 PM
We will be using similar metadata structure to opensea the only difference in ours is:
  • we have a mediaMimeType to explain what mime type the content is (video/music/image)
  • we have a content field which is the text for the publication
  • we have a media type for lens videos, music and images. image will be used for any NFT image but this media is a better name to have more then just an image
  • we have an appId probably renaming to source to allow you to tag sources for the metadata aka if you wanted to build an app and only query stuff with this source you will be able to
still a few final things to flesh it out but this is around about the standards we will be setting. Docs on all this will be coming soon. export interface Metadata { /** * The metadata version. */ version: string; /** * The metadata id so cid dont clash (could change to publication count but another db call!) */ id: string; /** * A human readable description of the item. */ description: Markdown; /** * The content of a publication. */ content?: Markdown; /** * This is the URL that will appear below the asset's image on OpenSea and others etc * and will allow users to leave OpenSea and view the item on the site. */ external_url?: Url; /** * legacy to support OpenSea will store any NFT image here. */ image?: Url; /** * Name of the item. */ name: string; /** * These are the attributes for the item, which will show up on the OpenSea and others NFT trading websites on the item. */ attributes: MetadataAttribute[]; /** * This is lens supported media type this can have images, videos and audio files. */ media?: Url; /** * This is the mime type of media */ mediaMimeType?: MimeType; /** * Legacy for OpenSea and other providers * A URL to a multi-media attachment for the item. The file extensions GLTF, GLB, WEBM, MP4, M4V, OGV, * and OGG are supported, along with the audio-only extensions MP3, WAV, and OGA. * Animation_url also supports HTML pages, allowing you to build rich experiences and interactive NFTs using JavaScript canvas, * WebGL, and more. Scripts and relative paths within the HTML page are now supported. However, access to browser extensions is not supported. */ animation_url?: Url; /** * This is the appId the content belongs to */ appId: AppId; } (edited)
